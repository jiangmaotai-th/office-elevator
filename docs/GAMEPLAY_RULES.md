# 核心玩法规则

本规则根据玩法说明 01–08 和 64 秒演示视频整理。代码实现和后续设计均以此文档为准。

## 一局游戏的阶段

### 1. 建设阶段

- 楼层类型与乘客目标在关卡开始前已经确定。
- 玩家先使用“放置井道”工具建立新的电梯井。
- 玩家使用“延伸井道”工具向上或向下延伸已有井道。
- 两个工具是不同操作，延伸井道不等于解锁新楼层。
- 玩家完成初始布局后点击“开始”，时间与乘客生成才正式运行。

### 2. 运营阶段

- 点击某一楼层，将该楼层加入所选电梯的停靠请求。
- 每层左侧使用独立颜色标记；乘客服装颜色与目标楼层颜色一致。
- 乘客身份统一为写字楼上班族，目的地不再依赖人物旁的数字标签。
- 同一电梯收到多个呼叫时，按呼叫先后顺序执行，每次点击都是一条独立停站指令。
- 停站队列允许重复楼层，例如依次点击 2、3、2、4 层，电梯会按自身速度自动执行 `2→3→2→4`，无需玩家继续点击。
- 电梯抵达楼层后先让到达目的地的乘客下车。
- 下车乘客按照固定间隔逐个离开轿厢，不会整批瞬间消失。
- 每一名乘客完成下车时独立播放一次音效，并显示本次停靠的递增送达计数。
- 玩家点击轿厢时，当前楼层所有能容纳的等待乘客上车，不限制方向。
- 玩家不点击轿厢时，只有目的方向与电梯刚才运行方向一致的乘客自动上车。
- 乘客按照队列逐个进入轿厢，不会在同一帧同时消失。
- 每一名乘客完成上车时独立播放一次上客音效。
- 上客队列完成前轿厢不会启动。
- 每层乘客严格按出现先后排队，队首最靠近电梯。
- 上客只能从队首开始，后来的乘客不能绕过前面的乘客插队。
- 电梯满员后，未上车乘客保持原顺序继续等待；前方乘客离开后队伍自动向前补位。
- 等待队列中的乘客会持续消耗耐心，满员不会暂停耐心计时。
- 耐心环表示已经等待的时间；前 50% 隐藏，等待过半后才出现并继续走满，剩余耐心进入最后 25% 时变红。
- 最后 25% 阶段，乘客所在楼层持续闪动并按节奏播放警报音。
- 任意一名等待乘客耐心耗尽并离开，本次运营立即失败。
- 失败面板提供“重新开始”，清空失败局的乘客、计时与电梯状态。
- 重新开始当前关卡时保留已建设楼层、金币与升级，只重置本局乘客、计时、送达和电梯状态。
- 主楼区域支持上下拖动浏览；楼层使用固定高度排列，增加楼层后可持续向上扩展，不再压缩重叠。
- 乘客上车后只显示目的需求，不会自动派发电梯。
- 无论轿厢里有哪些乘客，电梯都必须等待玩家下一次点击楼层后才移动。

### 3. 高峰阶段

- 高峰到来前显示倒计时、来源楼层类型和目标楼层类型。
- 玩家需要在高峰前调整井道和轿厢布局。
- 高峰期间对应路线会快速生成大量乘客。
- 高峰结束后可进入暂停建设或继续运营阶段。

### 4. 结算与升级

- 送达乘客获得金币并填充经验进度。
- 高耐心送达可获得 2X / 3X 徽章，提高经验或收益效率。
- 升级后获得金币与星星。
- 金币用于购买和延伸井道。
- 星星用于抽取运营升级，升级效果可以叠加。

## 电梯方向规则

每部电梯拥有独立状态：

- 当前楼层与连续位置
- 当前运行方向：上行、下行、空闲
- 当前目标楼层
- FIFO 停靠队列
- 轿厢乘客与容量
- 井道覆盖的最低和最高楼层

自动上客判断使用电梯抵达该层前的运行方向，不能使用抵达后已经变为空闲的状态。

示例：

- 电梯从 0 层上行抵达 2 层。
- 2 层有一名去 4 层的乘客和一名去 0 层的乘客。
- 未点击轿厢时，去 4 层的乘客自动上车，去 0 层的乘客继续等待。
- 点击轿厢时，只要有容量，两名乘客都会上车。

## 多电梯与井道

- 关卡中可以存在 S1、S2 等多条井道。
- 每条井道和轿厢独立运行。
- 点击楼层呼梯时，系统会把请求派给当前负载更低、距离更合适的电梯。
- 点击某个轿厢时，只让该轿厢在当前楼层按队列上客。
- 如果同层乘客多于单部电梯容量，第一部满员后，剩余乘客继续按先来后到等待，可由第二部电梯继续承接。
- 井道只能服务其覆盖范围内的楼层。
- 空中大堂解锁后，高层与低层电梯可在中转层换乘。

## 当前实现状态

已经实现：

- 写字楼楼层视觉、楼层色牌和服装颜色目的地识别。
- S1/S2 两部真实电梯并行运行，各自拥有位置、停站队列、载客、上客和下客队列。
- 默认高楼背景与可拖拽主游戏视窗，即使未增层也可以上下浏览。
- 楼层呼叫 FIFO。
- 手动点击轿厢无方向限制上客。
- 同方向乘客顺路自动上客。
- 耐心、送达、金币、倍率与基础升级循环。
