# 第一版美术资源审计

资源目录：`assets/电梯商场美术`

## 总体结论

当前方向已调整为现代写字楼与商务上班族，可以作为第一版美术基准。
目前多数文件仍是概念合集图，不应直接作为最终 Sprite 使用。接入时保留原图，
另行输出透明背景、边界规整、尺寸统一的生产资源。

## 可优先接入

### 写字楼主场景

- `电梯写字楼主场景图.png` 已确定楼层色牌、双井布局和商务空间基调。
- 运行时由代码动态绘制楼层、乘客和轿厢，不把效果图当成不可交互的整张界面。
- 右侧 S1 为当前可操作电梯，S2 作为后续多电梯功能的视觉预留。

### 旧商场主背景

- 视觉完成度高，可用于首页、关卡选择或模糊后的游戏底景。
- 不建议直接作为核心玩法楼层：图片中的店铺和楼层已经固定，无法匹配动态扩建、
  店铺替换和楼层解锁。
- 核心玩法应使用可重复拼接的楼层与商铺模块重新组合。

### 电梯素材

- 电梯外观与整体风格匹配，可以作为轿厢视觉基准。
- 需要重新导出：轿厢外框、内部、左门、右门、显示屏、上行灯、下行灯。
- 当前合集带大面积渐变底色，不能直接通过自动切图获得干净组件。

### 旧商场商铺

- 咖啡厅、美食街、服装店的辨识度和风格一致性良好。
- 当前文件没有 Alpha 通道，画面中的棋盘格已经烘焙进图片。
- 需要分别导出三张真正透明背景的商铺模块，并统一尺寸、楼板高度和出入口位置。

## 需要返工后接入

### 商务角色序列帧

- 人物造型可用，适合作为角色方向参考。
- `男上班族.png` 与 `上班族女性.png` 当前仍带烘焙背景，需要重新输出透明图。
- 正式角色必须提供可换色的上衣区域，服装颜色用于表示目标楼层。
- `年轻女性角色序列祯.png`、`时尚女性角色序列祯.png` 没有 Alpha 通道，
  棋盘格属于图片内容。
- 其他角色即使带 Alpha，也要检查透明区域是否真实透明。
- 同一动作内人物中心、脚底和身体比例变化较大，直接播放会产生抖动。
- 建议每个角色第一版只输出：
  - 左行 4 帧
  - 右行 4 帧
  - 正面等待 2 帧
  - 正面焦急 2 帧
  - 正面开心 2 帧
- 每帧固定为 `256 x 256`，脚底中心固定在同一坐标。

### UI 与弹窗

- `ui.png` 可作为布局和配色参考，但它是完整展示板，不能直接切成正式 UI。
- `弹窗套件.png` 没有 Alpha 通道。
- 正式 UI 应分别导出九宫格面板、按钮三态、进度条底和填充、标签底板及弹窗装饰。
- 所有文字和数字继续由 Cocos Creator 的 Label 渲染。

### Icon 合集

- 图形辨识度良好，但合集有渐变背景和光晕。
- 每个 Icon 应单独导出为透明 PNG，建议原始尺寸 `256 x 256`。
- 图标主体在画布内占比保持约 70%，不要保留外部展示板背景。

## 建议目录

```text
assets/
  art-source/                 # AI 原始图，仅作为源文件和参考
  resources/art/
    backgrounds/
    buildings/
    elevators/
    characters/
    icons/
    ui/
```

正式运行资源放入 `resources/art`，源图不直接参与首包构建。

## 第一轮生产资源

按以下顺序准备和接入：

1. 一个透明的空白楼层模块。
2. 写字楼大厅、办公区与会议区三个透明楼层模块。
3. 一套拆分后的电梯轿厢与左右门。
4. 两名角色的等待、左右行走、焦急和开心动画。
5. 六套楼层色牌、可换色服装材质与耐心状态图标。
6. 顶部 HUD、通用按钮和升级三选一卡片。

完成这六项后，即可把当前 Graphics 原型替换为第一版正式画面，同时保留动态楼层、
乘客和升级玩法。
